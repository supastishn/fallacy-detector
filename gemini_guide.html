Setting Up Google AI Studio (OpenAI Compatibility)

Google AI Studio provides access to Gemini models, and you can use its API with an OpenAI-compatible endpoint. This allows you to use Gemini models with this application.

Privacy Warning: Google may use data submitted to Gemini models via the API to improve their products. Please review Google's Terms of Service and Gemini API Terms for more information. Do not send sensitive or confidential information.

Step 1: Access Google AI Studio & Get an API Key

  • Go to Google AI Studio.
  • Sign in with your Google account.
  • Once in AI Studio, look for an option like "Get API key" (often found in the left sidebar or under a "Get started" section).
  • Click on "Create API key in new project" or select an existing project.
  • Copy the generated API key. Treat this key like a password and keep it secure.

Step 2: Configure in This Application

  • Go to the API Key Settings page in this application.
  • Paste the copied Google AI Studio API key into the "API Key" field.
  • In the "Base URL" field, enter: https://generativelanguage.googleapis.com/v1beta/openai
  • For the "Default Model" field, you can use one of the following examples (or other models compatible with the API key and base URL). Always verify the exact model identifier and its capabilities in your Google AI Studio console or the official Gemini documentation:
    • gemini/gemini-2.0-flash
    • gemini/gemini-2.5-flash-preview-05-20
    • gemini/gemini-2.5-flash-preview-04-17

    Note: The exact model name format required by the API (e.g., models/gemini-1.5-flash-latest vs gemini-1.5-flash) can vary. Refer to Google's official Gemini API documentation for the precise identifiers.

  • Check the official Gemini models documentation for the most current list of compatible model names for the API. The model name you enter here must be exactly as expected by the generativelanguage.googleapis.com API.
  • Click "Save Settings".
  • You can use the "Test API Connection" button to verify your setup. Ensure the model name used for testing is compatible.

Important Notes:

  • Model Naming: The exact model names (e.g., gemini-pro, models/gemini-1.5-flash-latest) required by the API might differ from display names in AI Studio. Refer to Google's API documentation for the correct identifiers to use in the "Default Model" field.
  • Capabilities: Gemini models often support multimodal inputs (text and images). Ensure the model you select supports the type of input you intend to use.
  • Rate Limits: Free tiers of the Gemini API typically have rate limits (e.g., requests per minute or per day). The user mentioned specific limits: 500 requests/day for one model and 1500 requests/day for another. Check your Google AI Studio console or documentation for the limits applicable to your key and models.
  • Security: Your API key is stored in your browser's local storage. Clear application storage via browser settings if you are on a shared computer and have finished your session.

That's it! You should now be able to use the application with Google's Gemini models via their OpenAI-compatible endpoint.
